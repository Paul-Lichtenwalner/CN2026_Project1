Networking Concepts

Domain Name System (DNS)

The Domain Name System (DNS) is responsible for translating human-readable domain names into IP addresses that computers can understand. When a user enters a website address into a browser, the system must determine which server on the Internet is hosting that website.

For example, instead of remembering a numeric IP address, users can access this website using its domain name. DNS servers receive the request and return the correct IP address of the web server hosting the site.

This process is known as domain name resolution. Tools such as nslookup or dig can be used to view the DNS records associated with a domain name.

IP Addressing

Every device connected to the internet must have a unique Internet Protocol (IP) address. An IP address identifies the location of a device on a network and allows data to be routed correctly between systems.

There are two main versions of IP addressing used on the Internet:

  • IPv4 – A 32-bit address format written as four numbers separated by periods (example: 192.168.1.1).
  • IPv6 – A newer 128-bit address format designed to support the growing number of devices connected to the internet.

When a user visits a website, DNS returns the IP address of the web server. The browser then uses this address to establish a connection with the server hosting the website.

HTTP vs HTTPS

Hypertext Transfer Protocol (HTTP) is the protocol used for communication between web browsers and web servers. It defines how requests and responses are formatted and transmitted over the Internet.

HTTPS (Hypertext Transfer Protocol Secure) is the secure version of HTTP. HTTPS encrypts data using Transport Layer Security (TLS), ensuring that information transmitted between the client and server cannot easily be intercepted or modified by attackers.

Modern websites use HTTPS to protect sensitive data such as login credentials, personal information, and financial transactions. GitHub Pages automatically provides HTTPS for hosted websites, ensuring secure communication between users and the server.

How These Technologies Work Together

When a user visits this website, several networking technologies work together to deliver the page:

  1. The user enters the website address into their browser.
  2. DNS resolves the domain name to an IP address.
  3. The browser connects to the server using the IP address.
  4. Communication occurs using the HTTPS protocol.
  5. The server responds with the website content, which is displayed to the user.

This process demonstrates how DNS, IP addressing, and web protocols interact to make websites accessible across the global internet.
